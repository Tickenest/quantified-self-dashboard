# quantified-self-dashboard

A personal analytics platform that ingests a daily life log from Google Sheets and delivers a multi-agent AI system for habit analysis, scheduled briefings, and natural language queries. Built on AWS with Bedrock, Lambda, DynamoDB, and a React frontend.

---

## What It Does

A supervisor agent orchestrates four specialist agents — Fitness, Food, Learning, and Gaming — each with tool access to a DuckDB query layer over a Parquet dataset. The system delivers:

- **Daily and weekly briefings** generated by the full agent pipeline, stored in DynamoDB, delivered by email via SES, and displayed in the dashboard
- **Natural language chat** — ask anything about your data and the supervisor routes to the appropriate specialist agents
- **Recommendations** — suggest meals, books, exercises, and games based on your logged history
- **A React dashboard** with weight trend visualization, exercise breakdown, food pattern heatmap, and weight-vs-exercise correlation analysis

## Multi-Agent Design

The supervisor receives all requests and delegates to specialist agents running in parallel via `ThreadPoolExecutor`. Each specialist has tool access limited to its domain:

| Agent | Data Access | Responsibilities |
|---|---|---|
| Fitness | weight_trend, weight_moving_avg, exercise_entries, exercise_correlation, exercise_streak | Weight trends, exercise patterns, correlations |
| Food | food_entries | Qualitative dietary assessment, pattern detection |
| Learning | training_entries, books_finished, books_abandoned, book_current | Reading pace, training progression |
| Gaming | gaming_entries | Session patterns, variety, frequency |

The supervisor synthesizes specialist findings into a single coherent response. For chat queries, only relevant agents are consulted based on keyword routing. For briefings and recommendations, all four are consulted in parallel.

## Two Environments

This project ships as two fully independent deployments sharing identical code.

**Personal** — connects to a real daily life log Google Sheet. Private URL.

**Demo** — connects to a synthetic dataset for a fictional subject with 6+ months of plausible life log data. Safe to share publicly.

Both environments are provisioned separately via Terraform using per-environment backend configs and tfvars files. The demo Google Sheet and synthetic dataset (`data/quincy_everyman.csv`) are included in the repository.

## Features

- Multi-agent orchestration with parallel specialist execution
- Daily and weekly AI-generated narrative briefings via email and dashboard
- Natural language chat interface with keyword-based agent routing
- Cross-domain recommendations (meals, books, exercises, games)
- Weight trend chart with 7-day moving average overlay
- Exercise frequency breakdown by activity type
- Food pattern heatmap with qualitative health classification
- Weight vs. exercise correlation analysis
- Configurable time window (7d / 30d / 90d / all time)

- Responsive layout

## Setup

### Prerequisites

- AWS account with Bedrock model access enabled for Claude Haiku
- AWS CLI configured
- Terraform >= 1.5
- Docker Desktop
- Node.js / npm

### 1. IAM User

Create an IAM user named `quantified-self-dashboard-dev` with the following managed policies:

- AmazonS3FullAccess
- AWSLambda_FullAccess
- AmazonDynamoDBFullAccess
- AmazonAPIGatewayAdministrator
- AmazonEventBridgeFullAccess
- IAMFullAccess
- CloudWatchLogsFullAccess
- AmazonBedrockFullAccess
- AmazonEC2ContainerRegistryFullAccess

Configure an AWS CLI profile:

```powershell
aws configure --profile quantified-self-dashboard-dev
```

### 2. Terraform State Buckets

Create one state bucket per environment:

```powershell
aws s3 mb s3://quantified-self-dashboard-terraform-state --region us-east-1 --profile quantified-self-dashboard-dev
aws s3 mb s3://quantified-self-dashboard-demo-terraform-state --region us-east-1 --profile quantified-self-dashboard-dev
```

### 3. Google Sheets

Create a Google Sheet for your life log with columns: `Date`, `Weight`, `Exercise`, `Training`, `Reading`, `Gaming`, `Food`. Share it as "Anyone with the link can view" and note the CSV export URL.

For the demo environment, create a separate Google Sheet and load `data/quincy_everyman.csv` into it. You can do this manually (File → Import in Google Sheets) or use the included `seed_demo_data.py` script with Google Sheets API credentials (see instructions at the top of that file).

### 4. Configure Environment Files

Copy the example files and fill in your values:

```
terraform/terraform.tfvars.demo.example     → terraform/terraform.tfvars.demo
terraform/terraform.tfvars.personal.example → terraform/terraform.tfvars.personal
terraform/backend.demo.hcl                  → already configured
backend.personal.hcl                        → fill in your state bucket name
frontend/.env.demo.example                  → frontend/.env.demo
frontend/.env.personal.example              → frontend/.env.personal
```

### 5. Deploy

For each environment (`demo` and `personal`):

**Push container images first:**
```powershell
.\deploy_refresh.ps1 -Environment demo
.\deploy_query.ps1 -Environment demo
```

**Apply Terraform:**
```powershell
cd terraform
terraform init `-backend-config=backend.demo.hcl
terraform apply `-var-file=terraform.tfvars.demo
```

**Deploy remaining Lambdas:**
```powershell
.\deploy_agent.ps1 -Environment demo
.\deploy_briefing.ps1 -Environment demo
```

**Trigger initial data refresh:**
```powershell
aws lambda invoke --function-name qsd-demo-refresh --region us-east-1 --profile quantified-self-dashboard-dev response.json
```

**Build and deploy frontend:**
```powershell
cd frontend && npm install && npm run build:demo
cd ..
.\deploy_frontend.ps1 -Environment demo
```

### 6. SES

Verify your sender email address in Amazon SES before briefing emails will deliver.

## Notes

- The chat password gate is client-side only — the real abuse protection is the 120,000 daily token budget enforced in the agent Lambda
- Specialist agents run in parallel; total response time is typically 15–20 seconds for four-agent queries
- Scheduled briefings bypass the token budget — only chat and recommendations are gated
- Container Lambdas (refresh, query) require Docker images to be pushed to ECR before `terraform apply` will succeed
